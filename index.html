Introduction

This project for my User Interface and User Experience class involves creating an app with a "favoring function". A favoring function is just any function in an app that allows the user to create a sublist of "favorite" objects. In my case, I tried to create a music app where users can place their favorite songs in a favorites playlist. To design this app, I did a comparative analysis on other websites with favoring functions and determined which functionalities I wanted to implement. Furthermore, as part of the assignment, I implemented a couple ways for users to sort and filter the database of songs; so users can search for songs more easily.

Part 1: Comparative Analysis

Spotify link, Youtube link, Google Maps link

Legend:

  • Sort in alphabetical order: Can sort by an element (ex. title, author, etc.) of the searched objects in alphabetical order
  • Sort by recency: Can sort by the recency in which displayed elements were created or added to the group
  • Add favorites button: Has a button to add an item to "favorites" (method of adding to favorites is clear)
  • Search tool: Can filter or search through the dataset of elements using a searchbar tool.
  • Multiple playlists or save spots: Has functionality for multiple "favorites" sections users can save to; usually for organization
  • Suggested things to add: Provides suggestions for items to add to favorites based on previous selections
  • Filter by type: Can filter results by a certain label assigned to each item (ex. Spotify has song genres, Youtube has video subject matter, Google maps has categories for each location: like restaurants or gas stations)

From the comparison chart, it looks like Spotify and Youtube provide more options for users to filter and sort data for ease of adding to a list of favorites compared to Google Maps. We can conclude this from how both Spotify and Youtube provide suggestions for things to add to the users' favorites, while Google Maps does not. Furthermore, unlike Google Maps, Spotify offers additional alphabetical sorting of data and Youtube provides recency sorting of data. In my opinions, such features make it easier for users to find new items to add to their favorites, because suggestions may pique some users' interest and sorting allows data to be more organized, and thus, more easy for users to look at and select from. However, I do not think this means Google Maps is a "worse app", it just means that they prioritize the use of their favorites function less than Spotify and Youtube; which makes sense, considering both Youtube and Spotify exist for the sole purpose of presenting content, while Google Maps offers other (potentially more important) functionalities such as giving directions.

For my project, I decided to take more elements from Spotify, since I'm also trying to develop a music app, so I decided to create a system where users can sort songs by title or artist alphabetically. Furthermore, as all apps I compared had very intuitive "add to favorites" buttons, I decided to do the same. Lastly, I implemented a filter by genre because all the apps I looked at had some "filtering by type".

Note: I decided not to implement some things I noticed, such as the search bar and multiple playlists/save spots because I think it would be outside the scope of this assignment.

Part 2: Implementation

As shown in the image above, the designed app offers both sorting and filtering functionalities.
The leftmost image shows the list of songs being sorted alphabetically by title after the user clicks on the "sort" and "title" dropdown items.
The middle image shows the filtering functionality of the app. After the user navigates from the "filter" dropdown menu to the pop genre, then only pop songs are left in the list of songs to select.
The rightmost image shows that filtering can also stack. After adding an additional filter of keeping only sub-three-minute songs, there is only one song remaining in the list because the "pop" filter is stacked with the "<3 mins" filter.

Finally, the app allows users to add to their list of favorite songs by clicking the "add to favorites" button next to the songs they want to add. This functionality is shown above when the user adds "Don't Lose Sight" to their list. Furthermore, I made it so duplicate songs can't be added again, because it is less likely users will want to add the same song to the favorites list twice. However ideally, this should be made into an option for users, so if I could improve on the app, I would have a confirmation screen pop up when a duplicate song is added.

The above images show the interface on my phone 13 phone screen. When the screen is more narrow (when the screen width is smaller than a certain threshold), the interface changes such that the favorites list is no longer on the right of the screen, but is instead a dropdown menu for users to click on to see their favorite songs.

The above images show the interface on a tablet sized screen. You can see the two different interfaces (with the favorites section as a dropdown menu vs. to the right of the screen) when the tablet screen is tilted sideways instead of straight.

Part 3: Conclusions

Overall, I think the app turned out quite well, there were a few things that I wish I could have improved on given more time. One example, as mentioned previously, was a popup menu when users try to add duplicate songs. In addition to this, I would like to improve the vertical phone interface. When the screen is too narrow, it looks like the songs get very squished, and I think that could be solved with more tweeks to the narrower design. Finally, I realized users cannot remove items from the cart without reloading the page, which I think could get annoying if users have very long carts and accidentally add songs. I would love to implement a "remove from favorites" button in the future.
